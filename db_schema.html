transactional_session() — Atomik Tranzaksiya Oqimi
1
Session ochish
SessionLocal() chaqiriladi. Yangi DB ulanishi pool'dan olinadi.
with transactional_session() as db: # db — bu Session obyekti
2
Ish bajarish (yield)
Tranzaksiya doirasida ixtiyoriy CRUD operatsiyalar amalga oshiriladi. Hamma o'zgarishlar xotirada saqlanadi (flush).
order = ManufacturingOrder(product_id=1, quantity=100) db.add(order) # stock_on_hand yangilash product.stock_on_hand -= required_qty db.flush() # ixtiyoriy — DB'ga yozadi lekin commit qilmaydi
?
Natija: muvaffaqiyat yoki xato?
Context manager chiqishida exception bo'ldimi — tekshiriladi.

✓ Xato yo'q — COMMIT

db.commit()
# Barcha o'zgarishlar DB'ga
# yoziladi — atom holda

✗ Exception — ROLLBACK

db.rollback()
# Hech qanday o'zgarish
# saqlanmaydi — raise
4
Session yopish
finally bloki db.close() ni chaqiradi. Ulanish pool'ga qaytariladi.
finally: db.close() # har doim bajariladi
database.py — To'liq Kod
from contextlib import contextmanager from sqlalchemy.orm import Session, declarative_base, sessionmaker from sqlalchemy import create_engine engine = create_engine(DATABASE_URL, pool_pre_ping=True) SessionLocal = sessionmaker(autocommit=False, autoflush=False, bind=engine) Base = declarative_base() # FastAPI route dependency def get_db() -> Generator[Session, None, None]: db = SessionLocal() try: yield db finally: db.close() # Atomik tranzaksiya — xato bo'lsa rollback @contextmanager def transactional_session() -> Generator[Session, None, None]: db = SessionLocal() try: yield db db.commit() # muvaffaqiyatli tugasa saqlaydi except Exception: db.rollback() # xato bo'lsa bekor qiladi raise finally: db.close()
ACID kafolatlari:
Atomicity — commit yoki rollback, yarim holat yo'q
Consistency — FK, UniqueConstraint, NN buzilmaydi
Isolation — boshqa sessionlar commit bo'lmagan o'zgarishni ko'rmaydi
Durability — commit bo'lgach disk'ga yoziladi
